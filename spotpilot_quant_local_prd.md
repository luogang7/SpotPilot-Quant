# SpotPilot Quant 现货量化领航台 PRD

## 1. 文档定位

本文档按当前系统功能反向补充，描述 SpotPilot Quant（现货量化领航台）已经形成的本地数字货币现货量化验证工作台。系统由 FastAPI 后端、Vue 3 前端、本地 MySQL 仓储、交易所适配器、AI 多通道分析、风控引擎、交易执行和审计日志组成。

第一版交易范围严格限定为 `Spot 现货`：

- 只允许买入现货、卖出现有持仓、撤销现货挂单。
- 不支持合约、永续、杠杆、保证金、裸做空、强平价和资金费率收益计算。
- Live 模式默认关闭，必须同时开启全局实盘闸门和单交易所现货闸门。
- AI 只输出分析、评分、建议信号和过滤条件，不能绕过策略与风控直接下单。

## 2. 产品目标

系统帮助个人量化交易者在本地回答以下问题：

- 系统、交易所、AI 通道和行情数据当前是否正常。
- 策略为什么产生买入、卖出现有持仓、观望或撤单相关信号。
- 回测是否基于本地历史 K 线、手续费和滑点得出可信结果。
- AI 分析、新闻情绪和市场上下文是否支持或拦截交易。
- 风控为什么允许交易、降低仓位、禁止开仓或全部暂停。
- Dry-run 或小额 Spot Live 执行链路是否完整可追溯。

## 3. 目标用户

- 个人数字货币量化交易者。
- 有 Python、数据分析或交易 API 基础的投资者。
- 希望先本地验证策略、风控和执行链路，再谨慎接入小额现货实盘的用户。

## 4. 当前功能范围

### 已纳入当前系统

- Dashboard 总览。
- Market 行情与 K 线图。
- Strategy 策略实验室。
- Backtest 回测、历史行情同步和 JSON/CSV 导出。
- AI 分析、新闻情绪、结构化校验和 AI 建议信号。
- Trading 交易执行、Dry-run 校验、小额 Spot Live、撤单、手动平仓。
- Risk 风控中心。
- Logs 审计日志。
- Settings 设置：交易所账号绑定、AI 模型通道、数据、新闻源、通知、每日推送和安全检查。
- 顶部系统状态：交易所、AI 通道、行情延迟、语言和主题。
- 系统控制：暂停策略开仓、Kill Switch 持久化。

### 明确不包含

- 合约、永续、杠杆、保证金和裸做空交易。
- 多账户资管和复杂权限系统。
- 高频盘口做市和低延迟撮合。
- 本地大模型训练。
- 自动生成并上线新策略。
- 面向第三方用户的 SaaS 部署。

## 6. 核心业务规则

- 全链路只允许 Spot 现货动作：`buy`、`sell_existing`、`hold`、`cancel_order`。
- 实盘下单只允许 `buy` 和 `sell_existing`；撤单使用独立撤单接口；手动平仓使用卖出现有现货接口。
- Live 模式需要 `TRADING_MODE=live`、`LIVE_TRADING_ENABLED=true`，并且对应交易所 `*_SPOT_TRADING_ENABLED=true` 且凭证完整。
- Binance 和 OKX 私有接口只用于读取余额、持仓、挂单以及安全闸门允许后的现货下单/撤单。
- API Key 必须禁止提现权限，建议开启 IP 白名单；系统不提供提现入口。
- 数据延迟、行情异常、AI 不可用、风控触发、Kill Switch 或人工暂停时必须阻止新开仓。
- 默认不生成假资产、假持仓、假订单和假行情；无数据时显示空状态或明确错误。
- 所有策略信号、AI 分析、风控判断、订单动作和系统控制都需要形成审计日志。

## 9. Strategy 策略实验室

### 页面原型

```text
┌──────────────────────────────────────────────────────────────┐
│ 策略实验室                 生成策略信号 | 保存策略参数          │
├──────────────────────┬───────────────────────────────────────┤
│ 注册策略列表          │ 策略详情                               │
│ MA Cross              │ 启用/停用  模式  状态                  │
│ RSI Mean Reversion    │ 参数：窗口、阈值、网格、仓位             │
│ Grid Trading          │ 风控参数：最大仓位、止损、止盈            │
│ Breakout              │ 支持信号 / 回测 / Live                  │
│ Bollinger Bands       │ 最近信号与拦截原因                      │
│ MACD Trend            │                                       │
│ Trend Pullback        │                                       │
│ DCA                   │                                       │
│ Funding Rate Guard    │                                       │
│ AI Filter             │                                       │
└──────────────────────┴───────────────────────────────────────┘
```

### 当前策略清单

- `ma_cross`：快慢均线交叉，支持信号、回测和现货 Live。
- `rsi_mean_reversion`：RSI 超卖买入、超买卖出现有持仓。
- `grid`：震荡区间内现货网格低买高卖。
- `breakout`：突破近期高点并获得成交量确认时买入。
- `bollinger_bands`：触及布林带下轨买入，回到中轨或上轨卖出。
- `macd_trend`：MACD 金叉买入、死叉卖出。
- `trend_pullback`：趋势回踩后重新站上短均线买入。
- `dca`：固定 K 线间隔分批买入，仅回测，不支持 Live。
- `funding_rate_guard`：风险过滤预留，不产生订单，不参与回测。
- `ai_filter`：AI 结构化结果校验，不直接生成订单。

### 功能要求

- 策略配置支持启停、运行模式、参数和风控参数保存。
- 生成信号时只运行已启用且非 `risk_filter` 的策略。
- 策略信号只允许现货动作，不允许开空或加杠杆。
- 最近信号需要展示时间、币种、策略、动作、价格、原因、拦截方和关联编号。
- 保存策略参数必须写审计日志。

## 10. Backtest 回测

### 页面原型

```text
┌──────────────────────────────────────────────────────────────┐
│ 回测       同步历史数据 | 导出JSON | 导出CSV | 运行回测          │
├──────────────────────┬───────────────────────────────────────┤
│ 回测参数              │ 回测结果                               │
│ 币种 / 交易所 / 周期   │ 状态 / 消息                            │
│ 策略                  │ 总收益 最大回撤 胜率                    │
│ 开始日期 / 结束日期    │ 收益曲线                               │
│ 初始资金 手续费 滑点   │                                       │
├──────────────────────┴───────────────────────────────────────┤
│ 历史回测模拟成交：开仓 平仓 币种 方向 价格 盈亏 手续费 退出原因   │
└──────────────────────────────────────────────────────────────┘
```

### 功能要求

- 回测只基于本地历史 K 线；本地无数据时返回 `data_unavailable`，提示先同步历史行情。
- 历史行情同步需要 `REPOSITORY_BACKEND=mysql`，支持 Binance 和 OKX。
- 同步结果展示 fetched、inserted、updated、首尾时间和状态消息。
- 回测参数包括交易对、策略、交易所、周期、起止日期、初始资金、手续费和滑点。
- 结果展示状态、消息、总收益、年化收益、最大回撤、胜率、Profit Factor、交易次数、权益曲线和模拟成交明细。
- 支持导出 JSON 和 CSV 文件，用于人工复盘或后续 AI 分析。
- 回测结果是历史模拟，不应被展示为真实订单或真实成交。

## 11. AI 分析

### 页面原型

```text
┌──────────────────────────────────────────────────────────────┐
│ AI分析                                         刷新            │
├──────────────────────────────────────────────────────────────┤
│ 目标：大盘/单币种 | 交易对 | 交易所 | 动作筛选 | 状态筛选        │
├──────────────────────────────────────────────────────────────┤
│ AI建议信号统计：总数 有效 防御型 看多 平均置信度                │
├──────────────────────────────┬───────────────────────────────┤
│ 建议信号列表                  │ 信号详情 Tabs                   │
│ action/status/score/confidence│ Summary / Evidence / Quality    │
├──────────────────────────────┴───────────────────────────────┤
│ 新闻情绪 | 结构化结果 JSON | Payload 校验                       │
└──────────────────────────────────────────────────────────────┘
```

### 功能要求

- 支持大盘和单币种两种分析范围。
- AI 输入包括新闻情绪摘要、现货量价快照、波动率、RSI、数据延迟和完整性。
- 支持 Right Code Responses API，以及 OpenAI Compatible Chat Completions 风格的 DeepSeek、通义千问、Kimi、GLM、MiniMax、Ollama、OpenRouter、AIHubMix、硅基流动等通道。
- 三路 AI Proxy 按优先级调用；超时、HTTP 失败、非 JSON 或结构化字段校验失败时自动故障切换。
- 全部 AI 通道不可用时使用保守降级：`risk_level=high`、`allowed_direction=none`，并禁止新开仓。
- AI 结构化输出必须包含市场状态、情绪分、风险等级、事件风险、允许动作、置信度、provider、model、rationale 和 structured_payload。
- 系统把 AI 分析转换为 AI 建议信号，包含 action、status、score、confidence、horizon、entry range、stop_loss、target_price、invalidation、watch conditions、risk summary、catalyst summary、evidence、data quality 和 metadata。
- 提供结构化 Payload 校验入口，非法 JSON 或缺字段时必须返回保守高风险结果。

## 12. 新闻情绪

### 功能要求

- 可按大盘或单交易对拉取 RSS 新闻摘要。
- 默认新闻源包括 CoinDesk、Cointelegraph、Decrypt、The Block、Blockworks、CryptoSlate、BeInCrypto、NewsBTC。
- 新闻摘要输出 sentiment_score、sentiment_label、risk_level、event_risk、article_count、source_count、status、message、rationale 和 articles。
- 刷新新闻情绪会清理对应 AI 缓存，并记录审计日志。
- 新闻功能关闭或无文章时，不伪造新闻；AI 使用空上下文或保守上下文分析。

- Live 模式只允许买入或卖出现有现货；撤单和手动平仓使用单独操作。
- 手动平仓会按市价或限价卖出现有持仓数量，不涉及合约平仓。
- 撤单必须指定交易所、交易对和订单 ID。
- 所有订单结果保存到仓储，并带 correlation_id。
- 支持根据 correlation_id 查询策略信号、AI 分析、风控事件、订单和日志组成的决策链路。

### 功能要求

- 缺少余额来源时禁止开新仓，避免无资产上下文下盲目交易。

│ 通知：飞书 企业微信 Telegram Email Slack Discord                │

### 功能要求

- 设置摘要展示交易所、AI 通道、服务商模板、数据配置、新闻源、通知配置和安全检查。
- 交易所账号绑定支持 Binance 与 OKX；OKX 需要 Passphrase；可配置 sandbox 和 spot trading enabled。
- 交易所账号测试会使用当前表单或已保存凭证进行私有接口轻量探测，并返回成功、失败原因、延迟和余额资产数量。
- AI 通道配置支持 A/B/C 三个槽位、Provider、Base URL、Model、Priority、Enabled、API Format 和 API Key。

- 通知支持飞书、企业微信、Telegram、Email、Slack 和 Discord；每个渠道支持单独保存和测试。
- 每日推送支持 `SCHEDULE_ENABLED`、`SCHEDULE_TIME`、`SCHEDULE_TIMES`、`SCHEDULE_RUN_IMMEDIATELY`，并可手动触发日报推送。
- 保存配置会写回 `.env`，不得把真实密钥写入 `.env.example` 或日志。

## 18. 数据模型与仓储

### MySQL 表职责

- `strategy_configs`：策略配置。
- `strategy_signals`：策略信号。
- `market_candles`：行情 K 线缓存与历史数据。
- `balances`：资产余额。
- `positions`：现货持仓。
- `orders`：订单和模拟订单。
- `risk_rules`：风控规则。
- `risk_events`：风控事件。
- `ai_analyses`：AI 分析结果。
- `backtest_runs`：回测摘要。
- `audit_logs`：审计日志。
- `system_state`：暂停与 Kill Switch 状态。

### 仓储规则

- 默认 `REPOSITORY_BACKEND=mysql`。
- 显式切换到 `memory` 时只用于临时验证，重启后数据丢失。
- 系统不得用内存仓储伪造默认账户资产、持仓、订单或行情。

- 企业微信群机器人。

- 系统异常和关键配置变更。

### 每日推送内容

- 资产、持仓和订单摘要。
- 风控状态和风险事件。

## 20. 状态设计

每个页面都需要处理：

- `loading`：正在加载。
- `empty`：无数据，且不展示假数据。
- `normal`：正常展示。
- `warning`：存在延迟、外部依赖失败、风险接近阈值。
- `error`：接口错误、数据库不可用、交易所或 AI 请求失败。
- `disabled`：未配置凭证、功能关闭或当前模式不可用。
- `stale`：当前缓存与筛选条件不匹配，需要刷新。

## 23. 成功标准

- 用户可以本地启动前后端并查看真实公共行情或明确错误状态。
- 用户可以添加交易对、同步历史 K 线、选择策略并完成回测。
- 用户可以配置 AI 模型通道并测试连通性，AI 返回必须通过结构化校验。
- 用户可以看到策略信号、AI 分析、风控判断、订单动作和日志之间的完整链路。
- Dry-run 不发生真实下单，但必须经过与 Live 一致的 AI 与风控校验。
- 小额 Live 前系统能明确确认现货权限、全局闸门和交易所闸门。
- 当数据、AI、交易所或风控异常时，系统明确提示并阻止危险交易。
